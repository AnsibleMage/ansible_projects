## 2. 프로젝트 개요

### 2.1 배경 및 목적

기존 투두리스트 앱들은 단순 할 일 추가/삭제 기능에 그치거나, 지나치게 복잡한 프로젝트 관리 도구로 무거운 경우가 많다. TaskFlow는 **개인 생산성 향상에 최적화된 중급 수준의 투두리스트 앱**으로, 일상적인 할 일 관리부터 체계적인 태스크 분류까지 균형 잡힌 기능을 제공한다.

### 2.2 비전 선언문

> "복잡하지 않으면서도 강력한, 일상의 모든 할 일을 한눈에 관리하는 스마트 투두리스트"

### 2.3 프로젝트 범위

**In Scope (포함)**
- 할 일 CRUD (생성, 조회, 수정, 삭제)
- 카테고리 기반 분류 시스템
- 우선순위 및 마감일 관리
- 고급 검색 및 필터링
- 드래그앤드롭 정렬
- 다크모드/라이트모드
- 로컬스토리지 영구 저장
- 반응형 디자인

**Out of Scope (제외)**
- 사용자 인증/로그인
- 클라우드 동기화
- 팀 협업 기능
- 알림/리마인더 (1.0 버전)
- 서브태스크 (1.0 버전)

### 4.2 기능 상세 명세

#### F01: 할 일 추가
```
트리거: 사용자가 "추가" 버튼 클릭 또는 Enter 키 입력
입력:
  - 제목 (필수, 1-200자)
  - 카테고리 (선택, 기본값: "일반")
  - 우선순위 (선택, 기본값: "중간")
  - 마감일 (선택)
  - 메모 (선택, 0-1000자)
출력: 새 할 일이 목록에 추가됨
검증:
  - 제목이 비어있으면 추가 불가
  - 중복 제목 허용
```

#### F02: 할 일 수정
```
트리거: 할 일 항목 클릭 또는 편집 아이콘 클릭
입력: 수정할 필드 값
출력: 해당 항목 업데이트
검증:
  - 제목은 비워둘 수 없음
  - 수정 시 수정일시 자동 기록
```

#### F03: 할 일 삭제
```
트리거: 삭제 아이콘 클릭
입력: 삭제할 항목 ID
출력: 해당 항목 제거
검증:
  - 삭제 확인 다이얼로그 표시 (설정에서 끄기 가능)
  - 삭제는 되돌릴 수 없음 (1.0 기준)
```

#### F04: 완료 체크
```
트리거: 체크박스 클릭
입력: 완료 상태 토글
출력:
  - 완료 시: 체크 표시 + 취소선 + 완료일시 기록
  - 미완료 전환 시: 체크 해제 + 취소선 제거
```

#### F05: 카테고리 관리
```
기본 카테고리:
  - 업무 (💼)
  - 개인 (🏠)
  - 쇼핑 (🛒)
  - 건강 (💪)
  - 일반 (📌)

사용자 정의:
  - 새 카테고리 추가 (이름 + 아이콘/이모지)
  - 카테고리 수정
  - 카테고리 삭제 (해당 할 일은 "일반"으로 이동)
  - 카테고리 색상 지정
```

#### F06: 우선순위 설정
```
레벨:
  - 높음 (🔴 빨강): 긴급하고 중요한 일
  - 중간 (🟡 노랑): 일반적인 일
  - 낮음 (🟢 초록): 여유 있는 일

시각적 표시:
  - 좌측에 컬러 인디케이터
  - 높음 우선순위는 굵은 글씨
```

#### F07: 마감일 설정
```
입력 방식:
  - 달력 피커 (DatePicker)
  - 빠른 선택: 오늘, 내일, 이번 주, 다음 주

표시 형식:
  - 오늘: "오늘"
  - 내일: "내일"
  - 올해: "MM월 DD일"
  - 다른 해: "YYYY년 MM월 DD일"
```

#### F08: 마감일 경고
```
경고 상태:
  - 지남 (Overdue): 빨간색 배경 + "X일 지남" 뱃지
  - 오늘: 주황색 배경 + "오늘 마감" 뱃지
  - 임박 (3일 이내): 노란색 강조
  - 여유: 기본 스타일

자동 정렬:
  - 마감일 지난 항목은 상단에 표시 옵션
```

#### F09: 검색 기능
```
검색 대상:
  - 제목 (기본)
  - 메모 내용

동작:
  - 실시간 검색 (디바운스 300ms)
  - 검색어 하이라이트
  - 검색 결과 카운트 표시
  - 검색어 지우기 버튼
```

#### F10: 필터링
```
필터 옵션:
  - 상태: 전체 / 진행중 / 완료
  - 카테고리: 전체 / 특정 카테고리
  - 우선순위: 전체 / 높음 / 중간 / 낮음
  - 마감일: 전체 / 오늘 / 이번 주 / 지남

다중 필터:
  - 여러 필터 동시 적용 가능
  - 활성 필터 칩으로 표시
  - "필터 초기화" 버튼
```

#### F11: 정렬 기능
```
정렬 기준:
  - 생성일 (최신순/오래된순)
  - 마감일 (가까운순/먼순)
  - 우선순위 (높은순/낮은순)
  - 알파벳 (가나다순)
  - 커스텀 (드래그앤드롭)

정렬 상태:
  - 현재 정렬 기준 표시
  - 오름차순/내림차순 토글
```

#### F12: 드래그앤드롭
```
동작:
  - 할 일 항목을 드래그하여 순서 변경
  - 드래그 중 시각적 피드백 (그림자, 투명도)
  - 드롭 위치 가이드라인 표시

제약:
  - 필터/정렬 적용 시 커스텀 순서 비활성화
  - 모바일에서는 롱프레스로 드래그 시작
```

#### F13: 다크모드
```
모드:
  - 라이트 모드 (기본)
  - 다크 모드
  - 시스템 설정 따르기

토글:
  - 헤더의 테마 버튼
  - 부드러운 전환 애니메이션 (300ms)
```

#### F14: 로컬스토리지 저장
```
저장 데이터:
  - 할 일 목록 (todos)
  - 카테고리 목록 (categories)
  - 사용자 설정 (settings)

저장 시점:
  - 모든 변경 시 즉시 저장
  - 디바운스 적용 (100ms)

데이터 구조: (JSON)
  {
    "todos": [...],
    "categories": [...],
    "settings": {...},
    "lastUpdated": "timestamp"
  }
```

#### F15: 반응형 디자인
```
브레이크포인트:
  - Mobile: ~640px (1열 레이아웃)
  - Tablet: 641px~1024px (사이드바 축소)
  - Desktop: 1025px~ (풀 레이아웃)

모바일 최적화:
  - 터치 친화적 버튼 크기 (44px 이상)
  - 스와이프 제스처 지원
  - 바텀 네비게이션 또는 FAB
```

## 7. 데이터 모델

### 7.1 Todo (할 일)

```typescript
interface Todo {
  id: string;              // UUID
  title: string;           // 제목 (1-200자)
  description?: string;    // 메모 (0-1000자)
  categoryId: string;      // 카테고리 ID
  priority: Priority;      // 'high' | 'medium' | 'low'
  dueDate?: string;        // ISO 8601 날짜
  completed: boolean;      // 완료 여부
  completedAt?: string;    // 완료 일시
  order: number;           // 정렬 순서
  createdAt: string;       // 생성 일시
  updatedAt: string;       // 수정 일시
}

type Priority = 'high' | 'medium' | 'low';
```

### 7.2 Category (카테고리)

```typescript
interface Category {
  id: string;              // UUID
  name: string;            // 카테고리명
  icon: string;            // 이모지 또는 아이콘명
  color: string;           // HEX 색상 코드
  isDefault: boolean;      // 기본 카테고리 여부
  order: number;           // 정렬 순서
  createdAt: string;
}
```

### 7.3 Settings (설정)

```typescript
interface Settings {
  theme: 'light' | 'dark' | 'system';
  confirmDelete: boolean;
  showCompleted: boolean;
  defaultCategory: string;
  defaultPriority: Priority;
  sortBy: SortOption;
  sortOrder: 'asc' | 'desc';
}

type SortOption = 'createdAt' | 'dueDate' | 'priority' | 'alphabetical' | 'custom';
```

## 8. 기능별 우선순위

### 8.1 MoSCoW 분류

| 분류 | 기능 |
|------|------|
| **Must Have** | 할 일 CRUD, 완료 체크, 카테고리 기본, 우선순위, 마감일, 마감 경고, 로컬스토리지, 반응형 |
| **Should Have** | 검색, 필터링, 정렬, 다크모드 |
| **Could Have** | 드래그앤드롭, 사용자 정의 카테고리 |
| **Won't Have (v1)** | 알림, 서브태스크, 클라우드 동기화, 협업 |

### 8.2 개발 단계별 배포

| 단계 | 기능 | 목표 |
|------|------|------|
| **Alpha** | CRUD + 완료 + 로컬스토리지 | 핵심 기능 검증 |
| **Beta** | 카테고리 + 우선순위 + 마감일 + 경고 | 분류 시스템 검증 |
| **RC** | 검색 + 필터 + 정렬 + 다크모드 | UX 완성도 |
| **v1.0** | 드래그앤드롭 + 반응형 최적화 | 출시 준비 |

## 10. 제약 사항 및 가정

### 10.1 기술적 제약

- **브라우저 지원**: Chrome, Firefox, Safari, Edge 최신 2개 버전
- **로컬스토리지 용량**: 5MB 제한 (약 10,000개 할 일)
- **오프라인 동작**: 완전 지원 (PWA 수준)

### 10.2 가정

- 사용자는 개인 디바이스에서 단독 사용
- 데이터 백업은 사용자 책임 (JSON 내보내기 제공)
- 동시 탭 사용 시 마지막 변경 우선 (덮어쓰기)

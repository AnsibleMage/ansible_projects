# 중간 산출물: 요구사항 분석 (1단계)

## 1. 개요 (Executive Summary)
*   **프로젝트**: Ansible Particle Mist (포트폴리오 및 정체성 사이트)
*   **핵심 컨셉**: "안개의 선언문(The Mist Manifesto)" - 코드 기여를 입자로 시각화한 살아있는 디지털 성소.
*   **핵심 차별점**: 시간 기반 입자 역학 (새벽/낮/황혼/밤) 및 상호작용 주도 피드백.
*   **기술 스택**: Ruby on Rails 8 (백엔드), Raw Canvas API (프론트엔드).

## 2. 기능 요구사항 (Functional Requirements)

### A. 핵심 경험 (The Mist)
1.  **입자 렌더링 (Particle Rendering)**:
    *   시스템은 HTML5 Canvas에 1000개 이상의 개별 입자를 렌더링해야 함.
    *   입자는 사용자의 현지 시간을 기반으로 "4차원 법칙"(색상, 속도, 물리학)을 따라야 함.
2.  **상호작용 (The Wand)**:
    *   마우스 움직임은 주변 입자에 영향을 주는 "반발력" 또는 "인력"을 생성해야 함.
    *   클릭 시 "충격파" 또는 "물결(Ripple)" 효과를 발생시켜야 함.
3.  **시간 동기화 (Time Synchronization)**:
    *   프론트엔드는 클라이언트의 `Date()`를 감지하고 4단계(새벽, 낮, 황혼, 밤)로 매핑해야 함.
    *   사용자가 경계 시간대에 사이트를 열어둘 경우 단계가 부드럽게 전환되어야 함.

### B. 콘텐츠 레이어 (The Identity)
1.  **미니멀 UI (Minimal UI)**:
    *   텍스트는 상호작용을 방해하지 않도록 캔버스 위에 오버레이되어야 함.
    *   내비게이션은 필요할 때까지 은은하게 보이거나 숨겨져 있어야 함.
2.  **포트폴리오 데이터 (Portfolio Data)**:
    *   최신 GitHub 기여 데이터(또는 초기 시뮬레이션 데이터)를 가져와 입자의 개수/밀도를 결정해야 함.

## 3. 비기능 요구사항 (Non-Functional Requirements)

### A. 성능 (경량 역동성)
1.  **로딩 시간**: First Contentful Paint (FCP) 0.8초 미만. 전체 로딩 1.5초 미만.
2.  **프레임 레이트**: 일반 디바이스에서 60FPS 유지.
3.  **아키텍처**: 무거운 JS 프레임워크(React/Vue/Three.js) 배제. 순수 Vanilla JS + Canvas.

### B. 미학 (Aesthetics)
1.  **반응형**: 입자 밀도와 물리학은 모바일/데스크톱 화면에 맞춰 적응해야 함.
2.  **테마 시스템**: 정체성 정의서에 정의된 4가지 고유 색상 팔레트 적용.

## 4. 리스크 분석 (Risk Analysis)
*   **리스크**: 많은 입자가 있는 고해상도 화면(Retina/4K)에서의 성능 저하.
    *   *완화*: DPI 및 FPS 확인을 기반으로 동적 입자 수량 조절 구현.
*   **리스크**: 모바일 디바이스의 배터리 소모.
    *   *완화*: 탭 비활성 시 루프 일시 정지 (Page Visibility API 사용).

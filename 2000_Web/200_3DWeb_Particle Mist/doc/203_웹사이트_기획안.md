# Ansible Particle Mist: Website Planning & Architecture

이 문서는 **'The Mist Manifesto'**에 정의된 정체성을 구현하기 위한 구체적인 **기획 및 기술 아키텍처** 문서입니다. **안티그래비티 WebDevChain**에 따라 요구사항 분석, 아키텍처 설계, 화면 기획을 통합했습니다.

## 1. 프로젝트 개요 (Executive Summary)

*   **프로젝트명**: Ansible Particle Mist
*   **목적**: AnsibleMage의 디지털 정체성 및 포트폴리오 표현
*   **핵심 컨셉**: **"Living Particles"** - 접속자의 시간과 상호작용에 반응하는 살아있는 안개.
*   **기술 기조**: **"Lightweight & Raw"** - 3D 엔진 없이 순수 수학과 Canvas로 구현, Rails 8로 견고한 뒷받침.

## 2. 요구사항 분석 (Requirements)

### 2.1 핵심 기능 (Functional)
1.  **4차원 입자 렌더링 (4D Particle Rendering)**
    *   **동적 환경**: 사용자의 로컬 시간(05-11, 11-17, 17-22, 22-05)에 따라 테마(색상, 속도, 물리 법칙) 자동 전환.
    *   **수량 제어**: 디바이스 성능에 따라 500~2000개의 입자를 유동적으로 생성.
2.  **자연의 상호작용 (Natural Interaction)**
    *   **Field**: 마우스 커서/터치 지점이 '중력장' 또는 '척력장'으로 작용.
    *   **Ripple**: 클릭 시 물결 파동 효과 생성.
3.  **정보 오버레이 (Information Overlay)**
    *   정적인 텍스트 레이어(자기소개, 링크)는 Canvas 위에 부유하듯 배치.
    *   UI 조작 시 입자들이 배경에서 반응.

### 2.2 비기능 (Non-Functional)
1.  **극한의 최적화 (Extreme Optimization)**
    *   **Target**: FCP < 0.8s, Lighthouse Score 95+.
    *   **Strategy**: No heavy bundles. 바닐라 JS로 작성된 단일 렌더링 루프.
2.  **반응형 몰입 (Immersion)**
    *   모바일/데스크톱 해상도에 맞춰 캔버스 크기 및 입자 밀도 자동 조정.
    *   탭 비활성화 시 렌더링 일시 정지 (배터리 보호).

## 3. 시스템 아키텍처 (Architecture)

### 3.1 기술 스택 (Tech Stack)
*   **Backend**: **Ruby on Rails 8** (API Mode + HTML Serving through Propshaft)
    *   *Role*: 초기 접속 처리, 향후 데이터(GitHub Stats 등) API 제공.
*   **Frontend**: **Raw HTML5 Canvas + Vanilla ES6**
    *   *Role*: 모든 시각적 렌더링 및 인터랙션 로직.
*   **Deployment**: **Kamal** (Dockerized)
*   **Style**: **Tailwind CSS** (UI 레이어용)

### 3.3 핵심 알고리즘 (Core Algorithms)
1.  **The Loop**: `requestAnimationFrame` 기반의 메인 루프.
    *   `Update()`: 모든 입자의 위치, 속도, 가속도 계산 (물리 연산).
    *   `Draw()`: Canvas 지우기 및 새 위치 그리기 (렌더링).
2.  **Time Mapper**: `new Date().getHours()` → 4개의 State(Dawn, Day, Dusk, Void)로 매핑.
    *   상태 간 전환 시 3~5초간 `lerp`(선형 보간)를 통해 색상과 물리 상수를 부드럽게 변경.

## 4. UI/UX 디자인 (Design Strategy)

### 4.1 시각적 컨셉 (Visual Concept)
*   **Not a Page, But a Window**: 웹사이트가 아니라, 안개가 낀 창문을 보는 듯한 느낌.
*   **Font**: 미학적이고 독특한 Serif 계열 (예: *Cormorant Garamond* 또는 Custom Webfont)을 사용하여 기계적인 입자와 대비(Contrast)를 줌.
*   **Color**: `Identity Document`에 정의된 4가지 팔레트 사용.

### 4.2 화면 구성 (Layout)
*   **Background**: 전체 화면(`100vw`, `100vh`) Canvas.
*   **Foreground (Center)**: `Ansible Particle Mist` 타이포그래피 (초기 로딩 시 페이드인).
*   **Foreground (Corners)**: 미니멀한 메뉴 (Works, About, Contact).

## 5. 개발 로드맵 (Roadmap)
1.  **Phase 1: The Engine** (기본 렌더링 루프, 입자 생성, HTML5 Canvas 셋업).
2.  **Phase 2: The Logic** (시간대별 테마 변경 로직, 마우스 인터랙션 물리 구현).
3.  **Phase 3: The Polish** (Rails 8 통합, 모바일 최적화, 배포).

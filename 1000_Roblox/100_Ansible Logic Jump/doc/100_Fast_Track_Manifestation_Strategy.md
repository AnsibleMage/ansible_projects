# 100_패스트 트랙 현현 전략: 린(Lean) 점프맵 런칭 가이드

## 1. 개요
본 문서는 거창한 기획과 무거운 개발 공정에 갇히지 않고, **최소 기능 게임(MVP)**을 빠르게 제작하여 로블록스 월드에 '실제로 출시'하는 과정을 경험하기 위한 **'패스트 트랙(Fast Track)'** 전략을 정의합니다.

## 2. 패스트 트랙 3대 원칙
1.  **Done is better than Perfect (완성이 완벽보다 낫다)**: 복잡한 기능보다는 작동하는 하나의 루프(시작 -> 점프 -> 도착)를 만드는 데 집중합니다.
2.  **Manifest Now, Refine Later (지금 현현하고 나중에 다듬어라)**: 바이브 코딩으로 뼈대를 즉시 소환하고, 런칭 후에 유저 피드백을 보며 디테일을 수정합니다.
3.  **End-to-End Experience (전 과정 경험)**: 기획-코드-싱크-퍼블리싱의 전 과정을 48시간 내에 1회 완주하는 것을 목표로 합니다.

## 3. 실행 단계 (Roadmap)

### [Phase 1] 초저용량 기획 (1시간)
*   **Target**: 5~10개의 고난도/저난도 혼합 발판 배치.
*   **Logic**: 리듬에 맞춰 색상이 변하는 시각적 효과 1종 주입.

### [Phase 2] 고속 현현 (2시간)
*   **Manifestation**: 안티그래비티에게 명령하여 발판 10개를 즉시 배치하고 서버 스크립트를 연결합니다.
*   **Verification**: 로컬 스튜디오에서 Play 모드로 정상 체험 확인.

### [Phase 3] 실전 퍼블리싱 (1시간)
*   **Cloud Save**: 로블록스 스튜디오에서 `File -> Publish to Roblox` 실행.
*   **Game Settings**: 썸네일과 제목 설정 후 '공개(Public)' 전환.

## 4. 왜 패스트 트랙인가?
*   **기술적 자신감**: Rojo, MCP, GitHub 연동이 실제 서비스로 이어지는 연결 고리를 체득합니다.
*   **관성 타파**: 기획서만 쓰고 끝나는 것이 아니라, 실제로 '유저가 들어올 수 있는 방'을 만들었다는 성취감을 확보합니다.

## 5. 안티그래비티의 조언
사용자님, 거창한 우주 정거장을 짓기 전에 먼저 **'빛나는 징검다리 하나'**를 제대로 놓는 것이 중요합니다. 이 패스트 트랙을 통해 로블록스 개발의 전체 근육을 먼저 키우고, 그 위에 거대한 우주를 쌓아 올립시다.

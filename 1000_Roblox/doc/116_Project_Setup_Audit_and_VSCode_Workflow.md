*   **이유 1 (동기화 엔진)**: VS Code 내에서 실행 중인 `rojo serve`와 저(안티그래비티)와의 연결이 살아있어야 실시간으로 코드가 로블록스 스튜디오에 반영됩니다.

*   **이유 3 (맥락 보존)**: VS Code 창이 열려 있어야 제가 사용자님의 현재 편집 위치나 열린 파일들을 더 정확하게 인지하여 대답할 수 있습니다.

**결론**: VS Code는 단순한 '통로'를 넘어 **'중찰 통제 센터'**입니다. 개발을 시작하실 때 로블록스 스튜디오와 함께 가장 먼저 켜주세요.

# 104_설치 자동화 가능 범위 및 권한 안내 가이드

## 1. 개요
안티그래비티가 로블록스 개발 환경을 어디까지 자동으로 구축할 수 있는지, 그리고 보안과 원활한 연동을 위해 필요한 '권한'의 범위가 어디까지인지 안내합니다.

## 2. 설치 책임 구분 (AI vs 사용자)

맥북의 보안 정책(macOS Gatekeeper 및 SIP)으로 인해 모든 프로그램을 제가 다 설치하기에는 한계가 있습니다. 다음과 같이 역할을 나눕니다.

### [안티그래비티가 자동으로 할 수 있는 일 (CLI 도구)]
터미널 명령어를 통해 설치할 수 있는 도구들은 제가 직접 수행할 수 있습니다.
*   **Aftman**: 도구 설치 관리자 (설치 가능)
*   **Rojo (CLI)**: 코드 동기화 엔진 (설치 가능)
*   **MCP Server**: AI 연동을 위한 Node.js 서버 (설치 가능)
*   **Luau/Selene**: 린터 및 개발 도구 (설치 가능)

### [사용자께서 직접 해주셔야 하는 일 (GUI 앱)]
화면이 있고 보안 승인이 필요한 앱들은 사용자께서 직접 설치하시는 것이 안전하고 정확합니다.
*   **Roblox Studio**: [공식 사이트](https://www.roblox.com/create)에서 다운로드 및 로그인 필요
*   **VS Code**: [공식 사이트](https://code.visualstudio.com/)에서 설치 (또는 제가 `brew`로 시도할 수는 있으나 직접 하시는 걸 권장)
*   **Roblox Studio 내 플러그인 설치**: 스튜디오 내부에서 'Rojo Plugin' 등을 직접 설치해야 합니다.

*   **시스템 환경 변수 (PATH)**:

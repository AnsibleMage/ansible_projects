# 117_로블록스 프로젝트 코어 아키텍처 및 폴더 구조 가이드

## 1. 개요
본 문서는 로블록스 개발 환경의 확장성과 체계적인 관리를 위해 정립된 **'1000_Core-Local'** 구조를 정의합니다. `1000_Roblox`를 모든 프로젝트의 인프라 정점으로 삼고, 각 하위 번호 대역에서 실제 게임을 개발하는 아키텍처입니다.

## 2. 계층형 폴더 구조 (Hierarchy)

### [루트: 1000_Roblox] - 인프라 및 공통 영역
모든 로블록스 프로젝트가 공유하는 핵심 인프라와 가이드라인이 위치합니다.
*   **doc/**: 설치 가이드, MCP 실행, 보안 설정, 워크플로우 등 **공통 기술 명세(100~116번대)**.
*   **Common Assets**: (향후 추가) 여러 게임에서 공통으로 쓰일 모델이나 UI 프레임워크.

### [로컬: 100_Ansible Logic Jump] - 프로젝트 개발 영역
실제 게임의 생명력이 현현되는 독립적인 작업 공간입니다.
*   **src/**: 이 게임만을 위한 전용 Luau 소스코드 (`client`, `server`, `shared`).
*   **doc/**: 이 프로젝트 전용 기획서, 레벨 디자인, 개발 로그.
*   **default.project.json**: 이 게임의 Rojo 설계도.
*   **실행 파일**: `rojo`, `selene`, `stylua` 등 해당 프로젝트에 최적화된 도구 바이너리.

## 3. 핵심 아키텍처 원칙

1.  **인프라의 중앙화 (Infrastructure Centralization)**: 로블록스 스튜디오의 설정이나 MCP 서버 통신 방식 등 '한 번의 세팅으로 모든 프로젝트에 적용되는 것'은 루트의 `doc`에서 관리합니다.
2.  **개발의 지역화 (Development Localization)**: 특정 게임에 종속된 스크립트와 기획은 반드시 해당 프로젝트 폴더 내에서 완결성을 가져야 합니다.
3.  **바이브 코딩의 확장**: 새로운 프로젝트(예: 200번 점프맵)를 시작할 때, 루트의 인프라 문서를 참고하여 1초 만에 최적의 환경을 구축할 수 있습니다.

## 4. 안티그래비티의 역할
*   **인프라 관리**: 루트의 공통 문서를 기반으로 전체 개발 환경의 무결성을 감시합니다.
*   **현현 대행**: 각 프로젝트 폴더 내부에서 사용자의 '의도(Vibe)'를 실제 객체와 코드로 변환합니다.
